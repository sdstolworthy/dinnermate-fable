# Dinnermate v2 Features Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** List membership & invites, walk/drive radius picker, restaurant hours display, and restaurant details (card flip + details page with map/reviews).

**Architecture:** Extends the v1 three-crate axum workspace + Flutter web client. One new migration (0002), one new provider-trait method (`details`), one new endpoint, membership gating on list writes. Client gains a flip animation, a details route with flutter_map, and a segmented radius control.

**Tech Stack:** unchanged from v1, plus client deps `flutter_map`/`latlong2`/`url_launcher`.

**Spec:** `docs/superpowers/specs/2026-06-12-v2-features-design.md` — READ IT FIRST. The v1 plan (`2026-06-11-dinnermate-v1.md`) documents the conventions; existing code is the ground truth.

```sql
CREATE TABLE list_members (
    list_id UUID NOT NULL REFERENCES lists(id) ON DELETE CASCADE,
```

### Core model additions (`dinnermate-core`)

```rust
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct HoursPeriod { pub day: u8, pub open: String, pub close: String } // day 0=Sun..6=Sat, "HH:MM"

// Restaurant gains (both default None; serde default for back-compat with seed JSON):
pub hours: Option<Vec<HoursPeriod>>,
pub utc_offset_minutes: Option<i32>,

#[derive(Debug, Clone, PartialEq)]
pub enum OpenStatus { Open { until: String }, Closed { opens_next: Option<String> }, Unknown }
// hours.rs:
pub fn open_status(hours: Option<&[HoursPeriod]>, utc_offset_minutes: Option<i32>, now_utc: DateTime<Utc>) -> OpenStatus

#[derive(Debug, Clone, PartialEq, Serialize, Deserialize, Default)]
pub struct ProviderDetails {
    pub website: Option<String>,
    pub phone: Option<String>,
    pub maps_url: Option<String>,
    pub reviews: Vec<Review>,
}
#[derive(Debug, Clone, PartialEq, Serialize, Deserialize)]
pub struct Review { pub author: String, pub rating: u8, pub text: String, pub relative_time: Option<String> }

// RestaurantProvider gains:
async fn details(&self, restaurant_id: &str) -> Result<ProviderDetails, ProviderError>;

// List model gains nothing; membership exposed via repo/service returns:
pub struct ListMembership { pub list: List, pub is_owner: bool }
```

`CoreError` gains `NotListMember` (→ 403 `NOT_LIST_MEMBER`) and `OwnerCannotLeave` (→ 422 `OWNER_CANNOT_LEAVE`).


### Details cache repo (`dinnermate-db`, new trait in core `repo.rs`)

```rust
#[async_trait]
pub trait DetailsCacheRepo: Send + Sync {
    async fn get(&self, restaurant_id: &str) -> Result<Option<(ProviderDetails, DateTime<Utc>)>, RepoError>;
    async fn put(&self, restaurant_id: &str, details: &ProviderDetails) -> Result<(), RepoError>; // upsert, fetched_at = now()
}
```

### `RoomService` addition

```rust
pub async fn restaurant_details(&self, code: &str, user_id: Uuid, restaurant_id: &str)
    -> Result<(Restaurant, ProviderDetails), CoreError>
```
Logic: room by code (404) → restaurant in deck (404 `RoomNotFound`? NO — restaurant absent → `UnknownRestaurant` mapped to **404** for this route; see API table) → cache get: fresh (<24h) → return; stale/missing → provider.details; on provider success put+return; on provider error → stale cache if any, else propagate. `RoomService::new` gains `cache: Arc<dyn DetailsCacheRepo>`.

### HTTP changes (all under `/api/v1`, OpenAPI updated in same commits)

| Method/Path | Req | Resp |
|---|---|---|
| `POST /lists/{code}/join` | — | 200 `{list, is_owner}` (idempotent) |
| `DELETE /lists/{code}/members/me` | — | 204; owner → 422 OWNER_CANNOT_LEAVE |
| `GET /lists` | — | 200 `{lists: [{...ListDto, is_owner}]}` owned+joined |
| `GET /lists/{code}` | — | 200 gains `is_member`, `is_owner` |
| `POST /lists/{code}/items` | unchanged | non-member → **403 NOT_LIST_MEMBER** |
| `GET /rooms/{code}/restaurants/{restaurant_id}/details` | — | 200 `{restaurant, website, phone, maps_url, reviews}`; unknown id → 404 `UNKNOWN_RESTAURANT` |

`RestaurantDto` gains `hours: [{day, open, close}] | null`, `utc_offset_minutes: int | null`.
Status-mapping note: `UnknownRestaurant` stays 422 on the swipe route; the details route maps it to 404 (resource lookup semantics) — implement via route-local match before the generic ApiError conversion.

### Google Places additions

- searchNearby field mask adds `,places.regularOpeningHours,places.utcOffsetMinutes`; map `regularOpeningHours.periods[].open/close.{day,hour,minute}` → `HoursPeriod` (skip malformed periods), `utcOffsetMinutes` → field.
- New Place Details: `GET {base_url}/v1/places/{restaurant_id}` headers `X-Goog-Api-Key`, `X-Goog-FieldMask: websiteUri,nationalPhoneNumber,googleMapsUri,reviews`; map → ProviderDetails (reviews: `authorAttribution.displayName`, `rating`, `text.text`, `relativePublishTimeDescription`; truncate to 5).

Dart models: `HoursPeriod`, `Review`, `RestaurantDetails{restaurant, website?, phone?, mapsUrl?, reviews}`; `Restaurant` gains `hours`/`utcOffsetMinutes`. `lib/src/hours.dart`: `OpenStatus openStatusFor(List<HoursPeriod>? hours, int? utcOffsetMinutes, DateTime nowUtc)` mirroring core's semantics exactly (same test table).
