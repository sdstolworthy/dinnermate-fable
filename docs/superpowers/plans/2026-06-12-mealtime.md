# Meal-time + OSM Timezones Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** "When are you eating?" room option filtering the deck to places open at that time, powered by tzf-rs timezone resolution for OSM restaurants.

**Architecture:** Core: `eat_at_utc` on RoomParams + filter rule reusing `open_status`. API: `tz.rs` (tzf-rs + chrono-tz) feeding `osm.rs` offsets; wire field; migration 0004. Client: segmented when-picker → UTC instant.

**Tech Stack:** + `tzf-rs`, `chrono-tz` (api crate only).

**Spec:** `docs/superpowers/specs/2026-06-12-mealtime-design.md`.

## Contracts

```rust
// core RoomParams gains:
#[serde(default)] pub eat_at_utc: Option<DateTime<Utc>>,
// filter::apply: eat_at Some && open_status(...,eat_at) == Closed{..} → exclude; Open/Unknown pass.
// api tz.rs:
pub fn utc_offset_minutes(lat: f64, lng: f64, at: DateTime<Utc>) -> Option<i32>
```
Wire: `eat_at` (ISO8601 UTC, nullable) on CreateRoomRequest + RoomDto. Migration 0004: `ALTER TABLE rooms ADD COLUMN eat_at TIMESTAMPTZ;`. Dart: `Room.eatAt DateTime?`, `CreateRoomRequest.eatAt`, when-picker per spec.
