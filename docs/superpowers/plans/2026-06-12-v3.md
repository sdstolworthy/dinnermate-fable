# Dinnermate v3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Room-from-list, OSM/Overpass provider (production default), room durability (recent rooms, participants, expiry), and PWA polish.

**Architecture:** Foundation first: restaurant data fields become `Option` (one migration, model + filter + UI ripple), which both OSM (no ratings) and list-decks (no coords) require. Then each feature lands on top. OSM I/O lives beside google.rs; hours parsing is a pure-core function.

**Tech Stack:** unchanged. Icons generated locally via rsvg-convert/ImageMagick (available on this box).

**Spec:** `docs/superpowers/specs/2026-06-12-v3-design.md` — READ FIRST. Existing code is ground truth; v1/v2 plans document conventions.

## Shared contracts (frozen)

### Core model changes

```rust
// Restaurant: these become Option (name, id, address(String, may be ""), photo_url unchanged):
pub cuisine: Option<String>,
pub price_level: Option<u8>,
pub rating: Option<f32>,
pub rating_count: Option<u32>,
pub lat: Option<f64>,
pub lng: Option<f64>,

// Room gains:
pub source_list_name: Option<String>,
```

Filter (unknown passes): cuisine/price/rating/radius each only exclude when the field is `Some` and fails the predicate. Ordering: `Some(rating)` before `None`, rating desc within Some, then name asc.

### `hours::parse_osm_opening_hours` (core, new file `osm_hours.rs`)

```rust
pub fn parse_osm_opening_hours(raw: &str) -> Option<Vec<HoursPeriod>>
```
Supported subset → `Some(periods)`: `24/7` (7 days 00:00–23:59); rules `;`-separated, each `Days Times`; Days = single (`Mo`) or range (`Mo-Fr`, may wrap `Sa-Su`? wrap means Sa,Su; also `Sa-Mo` wraps through Su,Mo) or comma list (`Mo,We,Fr`); Times = comma list of `HH:MM-HH:MM` (close < open allowed = crosses midnight). Day mapping Mo=1..Su=0 (HoursPeriod day 0=Sun). ANY unrecognized token (months, `PH`, `off`, `sunrise`, `+`) → `None` for the whole string. Whitespace-tolerant.

### `RoomService` additions

```rust
pub async fn create_with_deck(&self, user_id: Uuid, name: Option<String>, source_list_name: &str, deck: Vec<Restaurant>)
```

Synthesized stored params for list-rooms: `location_label = source_list_name`, lat 0.0, lng 0.0, radius_m 1000, cuisines [], price 1..=4, min_rating 0.0 (valid per existing validation; never used for filtering).
List-item → Restaurant mapping (route layer or a helper in core `list_deck.rs` — put it in core): `restaurant_id = item.source_restaurant_id.unwrap_or(format!("list-{}", item.id))`, name/cuisine/price/rating/address/photo copied (rating_count None, lat/lng None, hours None, offset None), order preserved.

### Migration `server/migrations/0003_v3.sql`

```sql
ALTER TABLE room_restaurants
    ALTER COLUMN cuisine DROP NOT NULL,
    ALTER COLUMN price_level DROP NOT NULL,
    ALTER COLUMN rating DROP NOT NULL,
    ALTER COLUMN rating_count DROP NOT NULL,
    ALTER COLUMN address DROP NOT NULL,

    ALTER COLUMN lng DROP NOT NULL;
ALTER TABLE rooms ADD COLUMN source_list_name TEXT;
```
(`address` nullable in DB; model keeps `String` — repo maps NULL→"".)

### HTTP changes

| Method/Path | Req | Resp |
|---|---|---|
| `POST /rooms/from-list` | `{list_code, name?}` | 201 `{room, deck}`; non-member 403 NOT_LIST_MEMBER; unknown list 404; empty list 422 INVALID_PARAMS |
| `GET /rooms/{code}` | — | gains `participants: [{display_name}]` (joined_at asc) and room gains `source_list_name` |

`RestaurantDto`: cuisine/price_level/rating/rating_count/lat/lng all nullable. `RoomDto` gains `source_list_name`. OpenAPI updated.
`RoomRepo` also gains `participants(room_id) -> Vec<Participant>` (joined_at asc) for the response.

### OSM provider (api crate, `osm.rs`)

`OsmProvider::new(http: reqwest::Client, base_url: String)` — base injectable for tests, prod default `https://overpass-api.de`. Config: `RESTAURANT_PROVIDER=osm`, optional `OVERPASS_URL`.
POST `{base}/api/interpreter`, form body `data=[out:json][timeout:10];nwr["amenity"="restaurant"]["name"](around:R,LAT,LNG);out center 60;`, header `User-Agent: dinnermate/1.0 (https://dinnermate.coolify.stolworthy.co)`.
Map per spec: id `osm-<type>-<id>`; cuisine tag first `;`-token lowercase `_`→space; address join of addr:housenumber/street/city (present parts, space+comma format "123 Main St, Maple City"); node lat/lon or `center`; hours via `parse_osm_opening_hours`; rating/price/photo/offset None. Entries with no resolvable coords are skipped. `details()` → `ProviderDetails { maps_url: Some("https://www.openstreetmap.org/<type>/<id>"), ..Default::default() }` (parse type/id back from the restaurant_id; malformed → default with None).

### Client contracts

- Dart `Restaurant`: cuisine/priceLevel/rating/ratingCount/lat/lng nullable; `Room` gains `sourceListName`; room detail record gains `participants: List<String>`.
- `ApiClient.createRoomFromList(String listCode, {String? name})` → `(Room, List<Restaurant>)` via POST /rooms/from-list.
- `RecentRooms` service (new `lib/src/recent_rooms.dart`): KeyValueStore-backed JSON under `dinnermate_recent_rooms`; `record(code, label)` (dedupe by code, most-recent-first, cap 5), `all()`, `remove(code)`.
